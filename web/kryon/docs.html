Kryon Documentation

Complete guide to using Kryon, a lightweight cross-platform UI toolkit with Plan 9-inspired filesystem control.

Installation

go get github.com/kryonlabs/kryon

Requirements

  • Go 1.25.7 or later (for building Kryon)
  • SDL2 for desktop platforms (Windows, Linux, macOS)
  • Sixel-compatible terminal for terminal UI (optional)

Note: While Kryon is written in Go, you can control the UI from any programming language through the 9P filesystem interface. No bindings needed!

Quick Start

package main

import (
    "github.com/kryonlabs/kryon/lib/kry"
)

func main() {
    // Create a new Kryon window
    kw, err := kry.CreateKryonWindow(wm, "My App")
    if err != nil {
        panic(err)
    }

    // Add a label widget
    label, _ := kw.AddWidget(kry.LabelWidget, "greeting")
    label.SetProperty("text", "Hello, Kryon!")

    // Keep window open
    <-make(chan struct{})
}

Platform Support

Kryon supports multiple platforms through pluggable rendering backends:

SDL2 Backend

Full-featured desktop GUI for Windows, Linux, and macOS. Provides hardware-accelerated graphics, input handling, and native window management.

Sixel Backend

Terminal-based graphics for Sixel-compatible terminals. Enables GUI applications in terminal environments without X11/Wayland.

Plan9Draw Backend

Native Plan 9 graphics support for authentic Plan 9 systems.

Headless Backend

In-memory rendering for server-side generation and testing.

Cross-Compilation

Use Go's standard cross-compilation to build for any target platform from any development machine.

Language-Agnostic Control

While Kryon is written in Go, the filesystem interface means you can control your UI from any programming language. Python, Rust, C, JavaScript, Bash — if it can read and write files, it can control Kryon. No language bindings needed!

Layout System

Row Layout

Arrange widgets horizontally with configurable gaps.

kw.SetLayout(kry.RowLayout, "gap: 10")

Column Layout

Arrange widgets vertically with configurable gaps.

kw.SetLayout(kry.ColumnLayout, "gap: 10")

Grid Layout

Arrange widgets in a 2D grid.

kw.SetLayout(kry.GridLayout, "columns: 3, gap: 10")

Absolute Layout

Manual pixel-perfect positioning.

widget.SetProperty("rect", "10 10 200 100")

Events

Event Handlers

    fmt.Println("Button clicked!")
})

    fmt.Println("Input changed:", value)
})

window.OnClose(func() {
    fmt.Println("Window closing")
})

Filesystem Interface

Reading Widget Properties

// Read via filesystem
value := readFile("/mnt/wm/win/1/wg/button/text")

Writing Widget Properties

// Write via filesystem
writeFile("/mnt/wm/win/1/wg/button/text", "Click me")

Event Files

Events are exposed as files that can be monitored for changes.

Examples

Simple Form

// Create form with label and input
label, _ := kw.AddWidget(kry.LabelWidget, "nameLabel")
label.SetProperty("text", "Name:")
label.SetProperty("rect", "10 10 100 30")

input, _ := kw.AddWidget(kry.TextInputWidget, "nameInput")
input.SetProperty("rect", "110 10 300 30")
input.SetProperty("placeholder", "Enter name")

Button with Click Handler

button, _ := kw.AddWidget(kry.ButtonWidget, "submit")
button.SetProperty("text", "Submit")
button.SetProperty("rect", "10 50 100 70")
button.OnClick(func() {
    value, _ := input.GetProperty("value")
    fmt.Println("Submitted:", value)
})
