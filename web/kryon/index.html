Kryon

Lightweight Cross-Platform UI Toolkit

Why Kryon?

Kryon is a lightweight cross-platform UI toolkit written in Go. Unlike heavy frameworks, Kryon has no runtime dependencies and compiles to a single binary. It brings Plan 9's elegant "everything is a file" philosophy to modern application development — control your entire UI through simple file I/O operations via 9P protocol.

Deploy to Windows, Linux, macOS via SDL2, use terminals with Sixel graphics, or run on Plan 9 systems. One codebase, multiple platforms — no platform-specific code needed.

Language-agnostic control: Since the UI is exposed as a filesystem, any programming language can control Kryon — Python, Rust, C, JavaScript, Bash, anything that can read and write files. No language bindings required!

Cross-Platform Support

Kryon runs everywhere Go runs, with multiple rendering backends for different environments:

  • Desktop: Windows, Linux, macOS via SDL2 backend
  • Terminal: Sixel-compatible terminals for terminal-based UI
  • Plan 9: Native Plan 9 systems via Plan9Draw backend
  • Server: Headless rendering for server-side generation

One codebase, multiple platforms — no platform-specific code needed.

Declarative Syntax

Define your UI with a simple, clean syntax. Kryon handles the rest.

window "My Form" {
    rect: 100 100 400 300

    label "Name:" {
        rect: 10 10 380 30
    }

    text-input nameInput {
        rect: 10 40 380 60
        placeholder: "Enter your name"
    }

    button submitBtn {
        rect: 10 70 100 90
        text: "Submit"
    }
}

Filesystem Interface

Control every aspect of your UI through the filesystem. Read and write widget properties as files.

/mnt/wsys/[id]/
├── mode              # Switch between "draw" and "wg" modes
├── label             # Window label/title (read/write)
├── rect              # Window geometry (read/write)
├── visible           # Window visibility (1 or 0)
├── kbd               # Keyboard input (both modes)
├── mouse             # Mouse input (both modes)
├── wctl              # Window control: hide, show, move, resize (both modes)
├── wg/               # Widget directories (wg mode only)
│   ├── nameInput/
│   │   ├── type      # "text-input"
│   │   ├── text      # Current value
│   │   ├── value     # Widget state
│   │   └── rect      # "10 40 380 60"
│   └── submitBtn/
│       ├── type      # "button"
│       ├── text      # "Submit"
│       └── rect      # "10 70 100 90"
├── text              # Text buffer (draw mode only)
├── cons              # Console output (draw mode only)
├── consctl           # Console control (draw mode only)
└── window            # Window image bitmap (draw mode only)

Lightweight by Design

Unlike heavy cross-platform frameworks, Kryon is:

  • No runtime dependencies: Single binary deployment
  • Small footprint: Minimal memory and CPU usage
  • Fast compilation: Go's build speed for rapid development
  • Modular: Use only the backends and widgets you need
  • No framework lock-in: Standard Go APIs and patterns

Architecture

Kryon (Cross-Platform UI Toolkit)
  ├── 89 widget types
  ├── Declarative UI syntax (.kry files)
  ├── State management system
  ├── Multiple rendering backends:
  │   ├── SDL2 (Windows, Linux, macOS)
  │   ├── Sixel (terminal graphics)
  │   ├── Plan9Draw (Plan 9)
  │   └── Headless (server-side)
  └── 9P filesystem control interface

Kryon's modular architecture lets you choose the rendering backend that fits your target platform.

Usage

Kryon is written in Go, making it easy to cross-compile for any platform:

# Build for current platform
go build -o wm ./cmd/wm

# Cross-compile for Windows
GOOS=windows GOARCH=amd64 go build -o wm.exe ./cmd/wm

# Cross-compile for macOS
GOOS=darwin GOARCH=amd64 go build -o wm ./cmd/wm

# Cross-compile for Linux ARM
GOOS=linux GOARCH=arm64 go build -o wm ./cmd/wm

Then use the Go API to create windows and widgets:

import "github.com/kryonlabs/kryon/lib/kry"

// Create Kryon window
kw, err := kry.CreateKryonWindow(wm, "My Window")

// Add widgets
label, _ := kw.AddWidget(kry.LabelWidget, "nameLabel")
label.SetProperty("text", "Name:")
label.SetProperty("rect", "10 10 380 30")

input, _ := kw.AddWidget(kry.TextInputWidget, "nameInput")
input.SetProperty("placeholder", "Enter your name")

button, _ := kw.AddWidget(kry.ButtonWidget, "submitBtn")
button.SetProperty("text", "Submit")
button.OnClick(func() {
    value := input.GetProperty("value")
    fmt.Println("Submitted:", value)
})
