Kryon

Filesystem-Based UI Toolkit

Why Kryon?

Kryon embraces the Plan 9 philosophy: everything is a file. Control your entire UI through filesystem operations — read widget values, write properties, and handle events through 9P. No complex APIs, just simple file I/O.

88 Widget Types

  • Input (24): text, password, checkbox, radio, dropdown, slider, toggle, etc.
  • Buttons (8): button, submit, icon, FAB, segmented, toggle button, etc.
  • Display (11): label, heading, image, progress bar, badge, separator, etc.
  • Containers (11): container, box, frame, scroll, split, tab, stack, etc.
  • Lists (8): list item, tile, view, grid, tree, chip, tag, card
  • Navigation (8): menu bar, toolbar, nav bar, breadcrumb, sidebar, etc.
  • Menus (4): dropdown menu, context menu, cascading menu, popup
  • Forms (4): form, field group, validation, input group
  • Windows (4): window, dialog, scaffold, fullscreen
  • Specialized (8): canvas, SVG, chart, color picker, calendar, spinner, slider, switch

Declarative Syntax

Define your UI with a simple, clean syntax. Kryon handles the rest.

window "My Form" {
    rect: 100 100 400 300

    label "Name:" {
        rect: 10 10 380 30
    }

    text-input nameInput {
        rect: 10 40 380 60
        placeholder: "Enter your name"
    }

    button submitBtn {
        rect: 10 70 100 90
        text: "Submit"
    }
}

Filesystem Interface

Control every aspect of your UI through the filesystem. Read and write widget properties as files.

/mnt/wm/win/[id]/kryon/
├── type              # Window type
└── wg/               # Widget directories
    ├── nameInput/
    │   ├── type      # "text-input"
    │   ├── text      # Current value
    │   ├── value     # Widget state
    │   └── rect      # "10 40 380 60"
    └── submitBtn/
        ├── type      # "button"
        ├── text      # "Submit"
        └── rect      # "10 70 100 90"

  • Column Layout: Vertical arrangement with configurable gaps
  • Grid Layout: 2D grid with configurable columns and rows

Architecture

Kryon (UI Toolkit)
  ├── 88 widget types
  ├── Layout managers
  ├── Event handling
  └── 9P filesystem interface

Kryon is a standalone, independent toolkit. Use it with any Plan 9-compatible system.

Usage

import "github.com/kryonlabs/kryon/lib/kry"

// Create Kryon window
kw, err := kry.CreateKryonWindow(wm, "My Window")

// Add widgets
label, _ := kw.AddWidget(kry.LabelWidget, "nameLabel")
label.SetProperty("text", "Name:")
label.SetProperty("rect", "10 10 380 30")

input, _ := kw.AddWidget(kry.TextInputWidget, "nameInput")
input.SetProperty("placeholder", "Enter your name")

button, _ := kw.AddWidget(kry.ButtonWidget, "submitBtn")
button.SetProperty("text", "Submit")
button.OnClick(func() {
    value := input.GetProperty("value")
    fmt.Println("Submitted:", value)
})
